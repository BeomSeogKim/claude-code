# 지식 관리 규칙

## 문서 생성 절차

1. 사용자 질문 또는 `/update-knowledge` 커맨드 수신
2. `knowledge/` 디렉토리에서 관련 기존 문서 검색 (Grep, Glob 사용)
3. 기존 문서가 있으면 → 갱신 절차로 이동
4. 없으면 → 공식 문서 리서치 → 새 문서 생성
5. 해당 카테고리의 `README.md` 인덱스 업데이트

## 문서 포맷

모든 지식 문서는 다음 구조를 따른다:

```markdown
# [제목]

> **요약**: 한두 줄 핵심 요약
> **출처**: 공식 문서 URL 또는 출처
> **신뢰도**: [VERIFIED] | [LIKELY] | [UNVERIFIED]
> **최종 갱신**: YYYY-MM-DD

## 본문

(핵심 내용)

## 핵심 포인트

- 포인트 1
- 포인트 2

## 메타

- 관련 문서: [[링크]]
- 태그: #tag1 #tag2
```

## 갱신 절차

1. 기존 문서 읽기
2. 최신 정보 fetch (WebSearch, WebFetch)
3. 차이점 비교
4. 문서 업데이트
5. 메타 섹션의 최종 갱신일 수정
6. 인덱스 업데이트

## 인덱스 관리

- 각 카테고리 디렉토리의 `README.md`가 인덱스 역할
- 문서 추가/삭제 시 반드시 인덱스도 함께 업데이트
- 인덱스에는 파일명, 제목, 한 줄 설명 포함
