# Claude 프롬프팅: 한국어 vs 영어 비교 분석

> **요약**: Claude Code 사용 시 한국어와 영어 프롬프팅의 효율성을 토큰 비용, 성능, 실용성 측면에서 종합 비교
> **출처**: Anthropic 공식 문서, 토큰화 분석 연구
> **신뢰도**: [VERIFIED]
> **최종 갱신**: 2026-02-16

## 핵심 결론

**영어 프롬프팅이 토큰 효율면에서 우수하나, 한국어도 실용적으로 충분히 사용 가능**

- **토큰 효율**: 영어 > 한국어 (영어가 더 효율적)
- **모델 성능**: 영어 100% 기준, 한국어 96.6~96.7% (Claude Sonnet 4.5/Opus 4.1)
- **권장**: 시스템 프롬프트/규칙은 영어, 사용자 대화/피드백은 한국어 혼용 가능

---

## 1. 토큰 효율성 비교

### 토큰화 특성

Claude의 tokenizer는 약 22,000개의 compact vocabulary를 사용하며, 언어별 지원 차이가 있음:

| 언어 | 토큰 커버리지 |
|------|---------------|
| 영어 | 가장 우수 (상위 10,000 영단어 중 8,311개가 단일 토큰) |
| 중국어 | ~1,100개 문자 토큰 |
| 한국어 | 수백 개 토큰 (정확한 수치 미공개) |
| 키릴 문자 | 수백 개 토큰 |
| 태국어 | 4개 토큰만 존재 |

**결론**: 영어가 토큰 효율이 가장 높음. 한국어는 중간 수준 지원.

### 실제 토큰 사용량 예시 (추정)

```
영어: "machine learning" → 2 tokens
중국어: "机器学习" → 3 tokens
한국어: "기계 학습" → 추정 3~4 tokens (공식 벤치마크 없음)
```

**비용 영향**:
- 같은 의미의 프롬프트를 한국어로 작성하면 영어 대비 **1.5~2배 토큰 소비** 예상
- 장기적으로 API 비용 증가 요인

---

## 2. 모델 성능 비교

### 공식 벤치마크 (MMLU 기준)

Claude의 다국어 성능을 영어 대비 상대 점수로 측정 (영어 = 100%):

| 모델 | 한국어 성능 | 영어 대비 격차 |
|------|-------------|----------------|
| Claude Opus 4.1 | 96.6% | -3.4% |
| Claude Sonnet 4.5 | 96.7% | -3.3% |
| Claude Sonnet 4 | 95.9% | -4.1% |
| Claude Haiku 4.5 | 93.3% | -6.7% |

**핵심 인사이트**:
- 한국어도 **영어의 96~97% 수준의 높은 성능** 유지
- Opus, Sonnet 4.5 계열에서 격차가 가장 작음 (3% 내외)
- 실무 사용에서 체감 차이는 미미할 수 있음

### 언어별 순위 (상대 성능 기준)

1. 스페인어 (98.2%)
2. 이탈리아어 (97.9%)
3. 프랑스어 (97.9%)
4. ...
9. 중국어 (96.9%)
10. **한국어 (96.7%)**
11. 일본어 (96.8%)

한국어는 주요 언어 중 **중상위권** 성능.

## 3. 실용적 고려사항

### 3-1. 영어 프롬프팅의 장점

- ✅ **토큰 효율 최상**: 비용 절감
- ✅ **공식 문서 호환**: 대부분의 예제/가이드가 영어
- ✅ **기술 용어 정확성**: "hook", "skill", "agent" 등의 개념을 번역 없이 사용
- ✅ **최고 성능**: 영어 기준 100% 성능

### 3-2. 한국어 프롬프팅의 장점

- ✅ **사고 자연스러움**: 모국어로 생각하고 지시 가능
- ✅ **빠른 반복**: 번역 오버헤드 없이 즉각 소통
- ✅ **문맥 표현력**: 미묘한 뉘앙스 전달 용이
- ✅ **협업 효율**: 한국어 팀원과 공유 시 이해 빠름

### 3-3. 하이브리드 접근 (권장)

**시스템 프롬프트 & 규칙**: 영어
- 이유: 재사용 빈도가 높아 토큰 효율 중요
- 예: `CLAUDE.md`, `.claude/rules/*.md`

**사용자 대화 & 피드백**: 한국어
- 이유: 즉각적인 소통 속도와 표현력 우선
- 예: 세션 대화, 질문, 수정 요청

**기술 용어**: 영어 유지
- 이유: 정확성과 공식 문서 일관성
- 예: hook, skill, agent, task, prompt

## 4. 비용 분석

### 시나리오별 토큰 소비 비교

| 시나리오 | 영어 토큰 | 한국어 토큰 (추정) | 비용 증가율 |
|----------|-----------|---------------------|-------------|
| 짧은 지시 (10단어) | ~10 | ~15-20 | +50~100% |
| 중간 프롬프트 (100단어) | ~100 | ~150-200 | +50~100% |
| 긴 문서 (1000단어) | ~1000 | ~1500-2000 | +50~100% |

**월별 비용 영향 예시** (가정: 일일 10만 토큰 사용):
- 영어만 사용: 100K tokens/day
- 한국어만 사용: 150-200K tokens/day
- 하이브리드 (시스템 영어, 대화 한국어): 120-150K tokens/day

**절감 팁**:
- Prompt caching 활용 시 **최대 90% 절감** 가능 (재사용 컨텐츠)
- 시스템 프롬프트는 영어로 고정하고 캐싱하면 비용 효율 극대화

## 5. 최종 권장사항

### 프로젝트 유형별 전략

| 프로젝트 유형 | 권장 접근 | 이유 |
|---------------|----------|------|
| 개인 학습 | 한국어 위주 | 이해 속도와 학습 효율 우선 |
| 프로토타입 | 한국어 위주 | 빠른 반복과 실험 중요 |
| 상용 서비스 | 영어 또는 하이브리드 | 비용 효율과 성능 최적화 |
| 팀 협업 | 하이브리드 | 팀 소통(한국어) + 시스템 효율(영어) |

### 구체적 가이드라인

1. **CLAUDE.md, rules, skills**: 영어로 작성
   - 재사용 빈도 높음 → 토큰 절감 효과 큼
   - 공식 예제 참고 용이

2. **세션 대화**: 한국어 자유롭게 사용
   - 3% 성능 차이는 실무 영향 미미
   - 사고 속도와 표현력이 더 중요

3. **기술 용어**: 영어 유지
   - hook (X: 훅, 갈고리)
   - skill (X: 스킬, 기술)
   - agent (X: 에이전트, 대리인)

4. **주석/문서**: 독자에 따라 선택
   - 팀 공유 → 한국어
   - 오픈소스 → 영어

---

## 핵심 포인트

- **토큰 효율**: 영어가 50~100% 더 효율적
- **성능 차이**: 한국어는 영어의 96.7% 성능 (실무 체감 미미)
- **비용**: 한국어만 사용 시 월 비용 50~100% 증가 가능
- **권장**: 시스템(영어) + 대화(한국어) 하이브리드 접근
- **Prompt caching**: 재사용 컨텐츠는 언어 무관하게 90% 절감 가능
